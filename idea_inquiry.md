# Idea Inquiry — AI 기반 학습 관리 서비스 (LeadMe)

> 최종 업데이트: 2026-04-09  
> 상태: ✅ LGTM — 다음 단계 진입 가능

---

## 아이디어 요약

공부를 시작하고, 지속하고, 복기할 수 있도록 돕는 AI 기반 학습 관리 서비스. 단순 계획 생성이 아닌 학습 지속성 유지와 실행 관리에 초점.

## 해결하려는 문제

학습자가 계획을 세워도 실행하지 못하거나, 중간에 포기하거나, 학습 과정을 돌아보지 않는 문제. "계획 → 실행 → 복기" 전체 루프를 관리해주는 서비스가 없음.

## 타겟 사용자

전 연령 대상. 학습 목표가 있지만 자기주도 학습에 어려움을 겪는 모든 사용자.

## 확정된 핵심 기능

| # | 기능 | 설명 | 우선순위 |
|---|------|------|---------|
| 1 | AI 기반 학습계획 생성 | 단계적 질문 수집 → 분량/구조 기반 계획 생성 (Macro Goal → Milestone → Todo) | P0 |
| 2 | 학습 노드 기반 칸반 시스템 | Todo Task를 Node로 관리, 각 Node에 Study Guide/Review/Tracker 포함 | P0 |
| 3 | 학습 상태 추적 및 피드백 | 정기 상태 기록, 집중 타이머, 피드백 리포트, 학습 속도 시각화 | P0 |
| 4 | 자기 관리 (Pre-check/Warm-up) | 학습 전 집중 상태 점검, 환경 정비, 동기부여 | P1 |
| 5 | 프로필 및 활동 시각화 | 학습 활동 매트릭스, 누적 공부 시간, 목표 달성 흐름 | P2 |

## 기술 의사결정

| 항목 | 결정 |
|------|------|
| 프론트엔드 | React + Zustand |
| 백엔드 | 추천에 위임 |
| DB | 추천에 위임 |
| AI 모델 | OpenRouter — google/gemma-4-26b-a4b-it:free |
| 인증 | Google 소셜 로그인 (MVP부터 로그인 필수, 게스트 모드 없음) |
| 플랫폼 | 웹 + 네이티브 앱 고려 |
| 배포 | Vercel |

## 사용자 답변 요약

- **Q1 (타겟 사용자)**: 전 연령
- **Q2 (기술 스택)**: 프론트엔드는 React + Zustand, 나머지는 추천에 따름
- **Q3 (인증)**: Google 소셜 로그인만. MVP에서도 게스트 모드 아닌 로그인 방식
- **Q4 (플랫폼/배포)**: 네이티브 앱까지 고려, 배포는 Vercel

## 구현 가능성 판단

- **현재 정보로 설계 시작 가능한 부분**: 핵심 기능 흐름, AI 연동 구조, 파라미터 수집 로직, MVP 우선순위, 기술 스택, 인증, 배포
- **나중에 결정해도 되는 부분**: 수익 모델, 프로필 페이지 상세, 알림 전달 방식, 동시 학습 계획 제한
