# 제품 요구사항 정의서 (PRD) — LeadMe

> 버전: 0.1 (spec 초안)
> 작성일: 2026-04-09

---

## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | LeadMe |
| 한 줄 설명 | 학습 시작, 지속, 복기를 AI가 관리해주는 학습 관리 서비스 |
| 핵심 가치 | 계획 생성이 아닌 **학습 지속성 유지와 실행 관리** |
| 타겟 사용자 | 학습 목표가 있지만 자기주도 학습에 어려움을 겪는 전 연령 사용자 |
| 플랫폼 | 웹 (반응형), 향후 네이티브 앱 고려 |

## 2. 기능 요구사항

### P0 — MVP 필수

#### FR-1: AI 기반 학습계획 생성

**사용자 스토리**: 학습자로서 나의 목표와 상황을 입력하면 실행 가능한 학습 계획을 받고 싶다.

**수락 기준**:
- [ ] 1차 필수 질문 7개를 순서대로 수집할 수 있다
- [ ] 필수 질문 완료 시 분량 기반 기본 계획(Macro Goal → Milestone → Todo)이 생성된다
- [ ] 2차 선택 질문 9개를 추가 입력할 수 있다
- [ ] 선택 질문 입력 시 구조 기반 정밀 계획이 생성된다
- [ ] 생성된 계획의 Milestone/Todo를 사용자가 검토하고 수정할 수 있다
- [ ] 확정된 Todo가 칸반 Node로 자동 생성된다

**파라미터 수집 구조**:

| 단계 | 질문 | 매핑 파라미터 | 필수 |
|------|------|-------------|------|
| 1차-1 | 무엇을 공부하려고 하나요? | `study_material.subject` | Y |
| 1차-2 | 최종 목표는 무엇인가요? | `final_goal` | Y |
| 1차-3 | 언제까지 달성하고 싶나요? | `deadline` | Y |
| 1차-4 | 하루에 어느 정도 시간을 쓸 수 있나요? | `available_time` | Y |
| 1차-5 | 현재 수준은 어느 정도인가요? | `current_level` | Y |
| 1차-6 | 어떤 소스를 기준으로 공부하나요? | `study_material.source.type` | Y |
| 1차-7 | 어떤 방식으로 관리받고 싶나요? | `management_style` | Y |
| 2차-1 | 소스의 이름은? | `study_material.source.name` | N |
| 2차-2 | 전체 분량/범위 | `study_material.source.total_volume` | N |
| 2차-3 | 추가 반영 조건 | `study_material.source.additional_info` | N |
| 2차-4 | 목차/커리큘럼/구조 정보 | `content_structure` | N |
| 2차-5 | 집중/약한 영역 | `focus_area` | N |
| 2차-6 | 학습 방식 선호 | `study_mode` | N |
| 2차-7 | 이번 주 구체적 목표 | `weekly_goal` | N |
| 2차-8 | 알림 빈도 | `notification_frequency` | N |
| 2차-9 | 동기부여 방식 | `motivation_focus` | N |

#### FR-2: 학습 노드 기반 칸반 시스템

**사용자 스토리**: 학습자로서 할 일 목록을 칸반 보드에서 시각적으로 관리하고, 각 할 일의 상세 학습 정보를 확인하고 싶다.

**수락 기준**:
- [ ] 칸반 보드에서 Todo/In Progress/Done 상태로 Node를 드래그 이동할 수 있다
- [ ] 각 Node 클릭 시 Node Page가 열린다
- [ ] Node Page에 Study Guide Section이 표시된다 (목표, 사전 지식, 생성 근거)
- [ ] Node Page에 Review Section이 표시된다 (회고 기록, 어려웠던 점, 보완점)
- [ ] Node Page에 Study Tracker가 표시된다 (집중 타이머, 상태 기록)

#### FR-3: 학습 상태 추적 및 피드백

**사용자 스토리**: 학습자로서 학습 중 집중 시간을 측정하고, 정기적으로 상태를 기록하여 AI 피드백을 받고 싶다.

**수락 기준**:
- [ ] 뽀모도로 타이머 (25분 집중 + 5분 휴식) 또는 일반 스톱워치를 사용할 수 있다
- [ ] 학습 중 상태 기록을 작성할 수 있다 (진행도, 방해 요인, 집중도)
- [ ] AI가 기록 기반 피드백 리포트를 생성한다
- [ ] 학습 속도 시각화 (예상 vs 실제 진행률)를 확인할 수 있다
- [ ] 집중도 저하 시 휴식 추천을 받을 수 있다

### P1 — 확장

#### FR-4: 자기 관리 (Pre-check / Warm-up)

**사용자 스토리**: 학습자로서 공부 시작 전 집중 상태를 점검하고 마음을 준비하고 싶다.

**수락 기준**:
- [ ] Pre-check 체크리스트를 완료할 수 있다 (정신 상태, 주변 정리, 소음 차단, 방해 요소, 일정)
- [ ] Warm-up에서 동기부여 문장 작성 또는 짧은 명상 타이머를 사용할 수 있다
- [ ] Pre-check 결과가 학습 세션에 연결되어 기록된다

### P2 — 후순위

#### FR-5: 프로필 및 활동 시각화

**사용자 스토리**: 학습자로서 나의 학습 활동을 시각적으로 확인하여 동기를 유지하고 싶다.

**수락 기준**:
- [ ] 학습 활동 매트릭스 (GitHub 잔디 스타일)를 볼 수 있다
- [ ] 날짜별 학습 패턴 차트를 볼 수 있다
- [ ] 누적 공부 시간을 확인할 수 있다
- [ ] 목표 달성 흐름을 시각적으로 추적할 수 있다

## 4. 사용자 여정 (주요 시나리오)

### 시나리오 1: 첫 학습 계획 생성 (Happy Path)

1. 사용자가 Google 로그인으로 진입
2. "새 학습 계획 만들기" 선택
3. AI가 1차 필수 질문 7개를 순서대로 진행
4. 기본 계획(Macro Goal → Milestone → Todo) 초안 생성
5. 사용자가 초안 검토 및 수정
6. 확정 시 칸반 보드에 Todo Node 자동 생성
7. (선택) 2차 질문으로 정밀화 후 계획 재생성

### 시나리오 2: 일일 학습 루틴

1. 홈에서 Pre-check 수행 (P1)
2. 칸반 보드에서 오늘 할 Node 선택 → In Progress로 이동
3. Node Page 진입 → Study Guide 확인
4. 집중 타이머 시작 (뽀모도로)
5. 타이머 종료 시 상태 기록 작성
6. 학습 완료 시 Review Section 작성
7. Node를 Done으로 이동
8. AI 피드백 리포트 확인

### 시나리오 3: 학습 복기 및 피드백

1. 누적된 학습 기록 기반으로 AI가 피드백 리포트 생성
2. 학습 속도 시각화 (예상 vs 실제)
3. 약점 분석 및 다음 행동 제안
4. 동기부여 메시지 (management_style에 따라 톤 조정)

## 5. 제외 범위 (Out of Scope)

| 항목 | 사유 |
|------|------|
| 수익 모델 (결제/구독) | MVP 이후 결정 |
| 소셜 기능 (그룹 학습, 팀 보드) | MVP 범위 밖 |
| 네이티브 앱 | 웹 반응형 우선, API 분리만 고려 |
| 다국어 지원 | 한국어 우선, 이후 확장 |
| 오프라인 모드 | 온라인 전용 |
| Google 외 소셜 로그인 | MVP에서 Google만 지원 |
| 푸시 알림 | 웹 알림/인앱 알림으로 대체 가능, 이후 결정 |
| 동시 학습 계획 제한 정책 | 이후 결정 |
