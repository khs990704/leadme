## 1. 문서 목적

본 문서는 AI 기반 학습 관리 서비스의 핵심 구조와 기능 범위를 정리한것이다.
**기획 설계 에이전트가 실제 설계 문서로 확장**할 수 있도록 구체화된 아이디어 설계 내용을 제공한다.

따라서 본 문서는 다음을 준수한다.

- **서비스의 핵심 흐름**이 명확할 것
- **기능 간 관계**가 분명할 것
- **AI의 역할과 한계**가 구체적으로 정의될 것
- **사용자 입력값과 시스템 생성 결과의 연결 구조**가 드러날 것

## 4. 핵심 기능

### 4.1. 자기 관리 (Supervise Your Mindset)

공부 전에 사용자의 집중 상태와 주변 환경을 정비하도록 돕는 기능이다.  
학습 계획 자체보다도, 실제 학습을 시작할 수 있는 상태를 만드는 데 목적이 있다.

#### 세부 기능

**Pre-check**
- 현재 정신 상태 점검
- 주변 정리 여부 확인
- 소음 차단 여부 확인
- 방해 요소 존재 여부 확인
- 다른 일정 유무 확인

**Warm-up (Optional)**
- 동기 부여 문장 또는 목표 기록
- 짧은 명상 타이머 제공

#### 기능 의도

이 영역은 부가 기능이 아니라,  
사용자가 학습을 시작하기 전 **실행 가능한 상태로 진입하도록 돕는 진입 장치** 로 본다.

---

### 4.2. 학습 노드 기반 칸반 시스템 (Create Kanban)

AI 가 사용자 목표를 기반으로 생성한 Todo Task를 Kanban의 Node 로 관리하고,  
**각 Node Page에서 개별 Task의 학습 진행을 추적**하는 기능이다.

각 Todo Task를 Node Page로 관리하는 이유는 각 Task의 수행/미수행 기록에 더하여
Task 실행에 필요한 Study Guide 제공,
Task 수행 이후 Review 이력 관리,
AI 기반 학습 진행 과정 추적 및 피드백
등을 제공하기 위함이다.

#### Node 페이지 구성

##### Study Guide Section
- 목표 설정
- 학습 전 필요한 사전 지식 목록
- 해당 Node 생성 근거
	- 분량 기반 생성인지
	- 목차/커리큘럼 기반 생성인지

##### Review Section
- 학습 종료 후 회고 기록
- 어려웠던 점
- 방해 요소
- 다음 학습 시 보완점

#### 학습 상태 추적 및 피드백 (Study Tracker)

정기적으로 사용자에게 Node에 상태 기록을 요청하고
사용자 입력 내용을 바탕으로 학습 상태를 추적한다.

##### 주요 기능
- 집중 타이머
	- 뽀모도로(Pomodoro) 기반 타이머 또는 일반 스톱워치 기능
- 정기적 상태 기록 요청 알림
	- 일정 시간(예: 1시간)마다 학습 진행도 체크 알림
	- 내부적(잡념, 졸음) / 외부적(스마트폰, 소음) 방해 요인 발생 시 즉각 기록
	- 집중도 저하 감지 시 휴식 시간 또는 스트레칭 추천
- 피드백 리포트 생성
- 학습 속도 시각화
- 휴식 시간 추천

### 4.3. AI 기반 학습계획 생성

이 영역은 사용자의 학습 목표를 구체화하고,  
실행 가능한 계획으로 전환하는 기능이다.

사용자의 목표를 구조화하고 지속 가능한 학습 흐름(Todo Task)으로 전환하는 것이 목적이다.

#### 4.3.1. 기본 원칙

- AI는 학습 대상을 전혀 모르는 상태에서도 기본 일정형 계획은 생성할 수 있다.
- 다만 학습 순서, 선행 개념, 실습 필요 여부, 회독 방식, 문제풀이 비중까지 반영한 정밀한 계획을 위해서는 추가 정보가 필요하다.
- 따라서 본 기능은 “정답형 계획 생성”이 아니라, 입력 정보 수준에 따라 점진적으로 정밀화되는 구조를 가진다.

#### 4.3.2. 파라미터 수집 구조

##### 파라미터 설계 원칙

사용자의 목표를 곧바로 todo로 변환하지 않고,  
질문 순서에 따라 필요한 정보를 단계적으로 수집한다.

질문은 많이 받는 것이 목적이 아니라,  
**빠르게 초안을 만들고 이후 정밀화하는 것** 이 목적이다.

따라서 수집 구조는 다음 두 단계로 나뉜다.

- 1차: 빠른 계획 생성을 위한 필수 질문
- 2차: 계획 정밀화를 위한 선택 질문

##### 최소 파라미터

###### study_material

- `subject`
	- 무엇을 공부하는지
		- 예: 영어, React, 정보처리기사, 알고리즘
- `source` (Object Array, 빈배열 허용, Object 추가 가능)
	- `source.type`
		- 학습 소스 종류
			- 예: book, lecture, exam, stack, web resource
	- `source.name`
		- 학습 소스 이름
			- Nullable
	- `source.total_volume`
		- 전체 분량 또는 범위
			- 예: 900페이지, 45개 챕터, 30강
			- Nullable
	- `source.additional_info`
		- 추가 반영 조건
			- 예: 하루 최소 1단원, 실습 위주 희망
			- Nullable

###### final_goal

- 최종 목표
- 예: 토익 850점, 프로젝트 완성, 시험 합격

###### deadline

- 목표 달성 기한
- 예: 3주, 2개월, 특정 날짜

###### available_time

- 하루 기준 공부 가능 시간
- 예: 하루 2시간, 평일 1시간 / 주말 3시간

###### current_level

- 현재 수준
- 예: 입문, 초급, 중급

###### management_style

- 관리 방식 선호
- 예: 부드럽게, 보통, 엄격하게

##### 권장 추가 파라미터

아래 정보가 있으면 AI는 단순 분량 배분을 넘어서  
학습 순서와 의미 단위를 반영한 정밀한 계획을 생성할 수 있다.

###### content_structure

- 학습 소스 구조 정보
- 예: 책 목차, 강의 커리큘럼, 시험 과목 구성, 프로젝트 단계

###### focus_area

- 집중하고 싶은 영역 또는 약한 영역
- 예: React hooks가 약함, 영어 듣기 위주로 하고 싶음

###### study_mode

- 학습 방식 선호
- 예: 개념 위주, 문제풀이 위주, 실습 위주, 회독 위주

##### 선택 파라미터

###### weekly_goal

- 주간 목표가 이미 존재하는 경우만 입력

###### notification_frequency

- 알림 빈도

###### motivation_focus

- 동기부여 방식
- 예: 칭찬형, 압박형, 목표 리마인드형

#### 4.3.3. 질문 순서 및 파라미터 매핑

##### 1차 질문 — 빠른 계획 생성을 위한 필수 입력

| 순서  | 질문                            | 매핑 파라미터                      |
| --- | ----------------------------- | ---------------------------- |
| 1   | 무엇을 공부하려고 하나요?                | `study_material.subject`     |
| 2   | 이 공부를 통해 이루고 싶은 최종 목표는 무엇인가요? | `final_goal`                 |
| 3   | 언제까지 달성하고 싶나요?                | `deadline`                   |
| 4   | 하루에 어느 정도 시간을 쓸 수 있나요?        | `available_time`             |
| 5   | 현재 수준은 어느 정도인가요?              | `current_level`              |
| 6   | 어떤 소스를 기준으로 공부하나요?            | `study_material.source.type` |
| 7   | 어떤 방식으로 관리받고 싶나요?             | `management_style`           |

**의도**
이 단계에서는 상세 정보가 부족하더라도  
기본적인 일정형 학습 계획을 먼저 생성할 수 있어야 한다.

##### 2차 질문 — 계획 정밀화를 위한 선택 입력

| 순서  | 질문                                        | 매핑 파라미터                                 |
| --- | ----------------------------------------- | --------------------------------------- |
| 8   | 공부하는 소스의 이름은 무엇인가요?                       | `study_material.source.name`            |
| 9   | 전체 분량이나 범위를 알고 있다면 입력해주세요.                | `study_material.source.total_volume`    |
| 10  | 추가로 반영했으면 하는 조건이 있나요?                     | `study_material.source.additional_info` |
| 11  | 가능하다면 목차, 커리큘럼, 과목 구성 같은 학습 구조 정보도 알려주세요. | `content_structure`                     |
| 12  | 특히 더 집중하고 싶은 부분이나 약한 부분이 있나요?             | `focus_area`                            |
| 13  | 어떤 방식으로 공부하고 싶은가요?                        | `study_mode`                            |
| 14  | 이번 주에 달성하고 싶은 구체적인 목표가 있나요?               | `weekly_goal`                           |
| 15  | 학습 알림은 얼마나 자주 받고 싶으신가요?                   | `notification_frequency`                |
| 16  | 어떤 방식의 동기부여가 학습에 도움이 되나요?                 | `motivation_focus`                      |

**의도**
이 단계에서는 계획 정밀도를 높이기 위한 정보를 수집한다.  
입력하지 않아도 기본 계획은 생성 가능해야 하며,  
입력될 경우 구조 기반 계획 생성으로 확장할 수 있어야 한다.

#### 4.3.4. 목표를 계층적으로 구체화 (상호작용 프로세스)

사용자와 AI가 단계적으로 협의하며, 막연하고 큰 목표를 실행 가능한 단위로 분해해 나가는 전체적인 흐름입니다. 한 번에 모든 계획을 확정하지 않고, 상위 목표부터 차례대로 검증하며 구체화합니다.

- **계층 구조**
    
    1. **Macro Goal:** 최종적으로 달성하고자 하는 큰 목표 (예: "정보처리기사 자격증 취득")
    2. **Milestone:** 중간 점검 지점 및 단계별 목표 (예: "필기 과목 1회독 완료", "기출문제 풀이")
    3. **Actionable Todo:** 실제 실행을 위한 일일/주간 단위의 학습 계획 (예: "1과목 소프트웨어 설계 1~2장 요약 읽기")
- **의도 검증 및 계획 확정**
    
    - 각 계층(특히 Milestone과 Todo)을 생성할 때, AI가 초안(Draft)을 먼저 제시합니다.
    - 사용자가 분량이나 배치가 적절한지 검토하고 수정(Interactive Edit)하는 단계를 거쳐 최종 확정합니다.

#### 4.3.5. 계획 생성 방식 (AI 동작 원리)

위의 계층 구조 중 **Milestone과 Todo를 생성할 때**, AI가 사용자의 입력 정보에 따라 내부적으로 계획을 수립하는 '기준'과 '원리'입니다.

- **1. 분량 기반 기본 계획 생성 (양적 분할)**
    
    - **조건:** 1차 질문의 기본 정보(총 학습 분량, 기간 등)만 제공되었을 때.
    - **원리:** AI가 학습 내용의 깊은 구조를 모르더라도, 일정과 분량을 기준으로 균등하게 목표를 분할합니다. 필요시 에빙하우스 망각곡선 등 범용적인 복습 주기를 일정에 기계적으로 배치하여 기본 계획을 생성합니다.
- **2. 구조 기반 정밀 계획 생성 (질적 분할)**
    
    - **조건:** 2차 질문을 통해 학습 매체의 세부 정보(목차, 커리큘럼, 선수 지식 등)가 제공되었을 때.
    - **원리:** 단순한 분량 나누기가 아닌, 개념의 '의미 단위'와 '선후행 학습 관계'를 분석합니다. 연관된 개념을 묶거나 난이도에 따라 일정을 배분하여, 훨씬 논리적이고 정밀한 구조 중심의 계획을 생성합니다.

### 4.4. AI 활용 계획

본 서비스는 초기 MVP 및 테스트 단계에서  
별도 모델 서버를 직접 운영하지 않고 외부 API 라우터를 활용한다.

#### 4.4.1. 사용 모델

- Platform: `OpenRouter`
- Model: `google/gemma-4-26b-a4b-it:free`

#### 4.4.2. 활용 목적

- 단계형 질문 인터랙션 처리
- 사용자 응답 기반 파라미터 구조화
- 학습 계획 초안 생성
- Macro Goal / Milestone / Todo 분해
- 사용자가 Node Page에 정기적으로 기록한 학습 과정을 바탕으로 학습 상태를 추적
	- 피드백 리포트 생성
	- 학습 속도 시각화
	- 휴식 시간 추천
	- 동기부여 메시지 생성
- 관리 스타일에 따른 응답 톤 조정

#### 4.4.3. 선택 이유

- 별도 GPU 서버 없이 빠르게 연동 가능
- MVP 단계에서 구현 및 교체가 용이함
- 초기 테스트 규모에서는 무료 모델 기반 검증이 가능함
- 기획서 기반 바이브코딩 및 빠른 프로토타이핑에 적합함

#### 4.4.4. 적용 원칙

- AI는 자유 생성형 응답보다 구조화된 출력 중심으로 사용한다.
- 질문 수집 단계에서는 정해진 질문 순서를 기반으로 필요한 파라미터를 수집한다.
- 계획 생성 단계에서는 JSON 또는 명확한 스키마 기반 응답을 우선한다.
- 향후 품질 이슈가 발생할 경우 동일 인터페이스 내에서 다른 모델로 교체 가능하도록 설계한다.

#### 4.4.5. 초기 역할 범위

**질문자**
필요한 정보를 순서대로 수집한다.

**구조화자**
사용자 응답을 파라미터 형태로 정리한다.

**계획 생성기**
수집된 정보를 바탕으로 기본 또는 정밀 계획을 생성한다.

**피드백 코치**
학습 기록 기반 피드백과 다음 행동을 제안한다.

---

### 4.5. 프로필 페이지

개인 활동 데이터를 시각화하는 영역이다.

**주요 요소**
- 학습 활동 매트릭스
- 날짜별 학습 패턴
- 목표 달성 흐름
- 누적 공부 시간

이 영역은 학습 기록의 열람과 동기 부여를 담당하지만,  
초기 MVP에서는 우선순위가 상대적으로 낮다.

## 6. 설계 에이전트 전달 시 핵심 해석 포인트

본 문서를 기반으로 기획 설계 에이전트가 후속 설계 문서를 생성할 때,  
특히 아래 사항을 중요하게 반영해야 한다.

### 6.1. 이 서비스의 본질

이 서비스는 “공부 계획을 예쁘게 짜주는 서비스”가 아니라  
**학습 시작, 지속, 복기를 관리하는 서비스** 이다.

### 6.2. AI의 역할

AI는 모든 것을 자유롭게 생성하는 주체가 아니라,  
정해진 질문 흐름에 따라 정보를 수집하고, 구조화하고, 계획을 정리하는 역할을 수행한다.

### 6.3. 계획 생성의 한계와 확장

AI는 학습 내용을 완전히 모르더라도 기본 계획을 생성할 수 있다.  
다만 구조 정보가 충분할수록 계획 품질이 높아진다.

### 6.4. 초기 제품 우선순위

초기 MVP는 아래 흐름이 살아 있어야 한다.

- 목표 입력
- 필수 질문 수집
- 기본 계획 생성
- Node 생성
- 학습 진행 기록
- 종료 후 리뷰
- 피드백 제공

그 외 통계 고도화나 부가 기능은 후순위로 둔다.
